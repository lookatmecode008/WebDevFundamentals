Purpose
This is the basic manual and guide to interacting with the website created for Lesego Public Primary School by Aaron M. Kotela in November 2025. It provides guidance regarding basic functions and possible edits for the user (or reader) to introduce. It consists of HTML, CSS and JavaScript that have been customised for both mobile and desktop use.

Toolbox
HTML: Used to construct the skeleton of the website as well as the container of all other languages
CSS: Used to apply styles and effects to the HTML. Stored internally in each HTML file under a <styles> inside the <head> tag. 
JS: Used mainly to apply interactivity and user responsiveness to the website (e.g. buttons and dropdowns). Stored internally in each HTML files under a <script> tag at the bottom of the <body> tag. Is intimately linked with CSS.
Font-awesome: Used as a source of icons in index.html. 
Github Account: Used to store, host, edit and update the website. Supports version control and multi-user management. A couple of notes:
		*Login credentials are available on the school google account.
		*Repo has been made public to support hosting.
		*Has version control so reverting to last function is possible.
Formspree: Used to handle the form used the website. Intimately linked to Contacts.html. Alerts form submissions via the school gmail account. Please refer to the site with matters of privacy and file retention.
Netlify: Used for hosting early versions of the website during testing phases. Supports domain purchases but more expensive than GoDaddy.com. A couple of notes:
		*Login credentials are available on the school google account.
		*There is a link to a beta version of the website that is still active, this was part of version control that could not be deactivated in time. 
		*Netlify is a free hosting service with a build-time limit per month, friendly user interface and linkable to the school Github repo.
Cloudflare: Used as the best substitute for hosting, domain-purchase and email-server. Not cheap but reliable and easy to maintain.

State and Recommendations
Active link
Netlify at https://lesegopps.netlify.app/ 
and Github at https://lesegopps.github.io/school-site/

The school currently has two active links from two hosting-service providers. Either has pros and cons, the first is easier to use and handles forms locally (so Formspree will not be employed) but it does not support version control. The latter is more complex, requires external form handlers and raises matters of security and privacy with its "public repo" demand yet offers arguably the best version control, automatic updates and multi-user code management. Of course it is possible to link the two but only the latter is recommended since it is far more beneficial than the former. The former was also only meant for testing phases and early features but barred unpublishing the website as build-time limits for the month were exceeded, this was reason enough to migrate to Github. At most, hybridising the two is beneficial to a better extent.

2. Security
The login credentials of each of the sites used are stored in the school's google account for convenience and relatively safe-keeping. These credentials are available upon request from the principal or head administration clerk. Login also uses Two-Factor Authentication so timely communication is essential. 
In order to host on Github, the repo privacy must be set to public. This means people can access and review the source code. Luckily, changes cannot be committed without the permission of the creator of the repo (i.e. the school Google account).  

2. Google Search Console (GSC)
The website has been submitted to Google Search Console using "https://lesegopps.github.io/school-site/". GSC is what allows the website to show up on google search results. Since crawling (Google essentially visits your site to see what is it about) was taking too long and a sitemap could not work (I honestly do not know why), individual pages of the website were submitted but there no were significant results at the time of this document. AI had suggested that it would take weeks before the site would appear on Google so it is recommended that the GSC progress be tracked regularly and updated as requested to ensure smooth sailing.

3. Tweaks and Touches
Insert at least two navigation links/buttons on each webpage, one of them MUST redirect to the home page (This also boosts GSC crawling).
